Presentación

Caracterización de la Asignatura

La asignatura de Desarrollo Full Stack capacita a los estudiantes en el diseño, implementación y gestión de aplicaciones web completas, abarcando desde la arquitectura cliente-servidor, desarrollo front-end accesible y responsivo con HTML5, CSS3 y frameworks modernos, hasta la aplicación de patrones de diseño en back-end y el uso de Angular, servicios RESTful, integración con Firebase, control de versiones, y despliegue continuo para crear soluciones escalables y modernas.

Intención Didáctica

Esta asignatura tiene como intención didáctica que el estudiante comprenda y aplique de manera integral los conceptos, herramientas y patrones de diseño necesarios para construir, implementar y mantener aplicaciones web completas, accesibles, responsivas y escalables, utilizando tecnologías modernas desde el front-end hasta el back-end, así como prácticas de control de versiones e integración continua.

Competencias a Desarrollar

Competencias Específicas

Diseñar e implementar interfaces web accesibles, responsivas y usables, utilizando tecnologías y frameworks actuales como HTML5, CSS3, lenguajes de lado servidor (PHP, JSP, ASP), Bootstrap y Angular, con base en principios de usabilidad y experiencia del usuario.

Implementar patrones de diseño en la lógica del lado servidor para construir arquitecturas sólidas, escalables y mantenibles.

Competencias Genéricas

  • Comunicación oral y escrita
  • Habilidad para trabajar en equipo
  • Habilidades de investigación
  • Habilidades en el uso de tecnologías de la información y comunicación

Competencias Previas Requeridas

  • Comprender los fundamentos de la programación orientada a objetos y funcional
  • Modelar sistemas mediante diagramas UML
  • Analizar y resolver problemas computacionales con algoritmos y estructuras de datos
  • Conocer fundamentos del desarrollo web (HTML, CSS, JavaScript)
  • Tener nociones básicas sobre bases de datos y operaciones CRUD

Temario

1. Fundamentos del Desarrollo Full Stack

  • Rol y diferencias del Front-End, Back-End y Full Stack
  • Arquitectura cliente-servidor y SDN en capas
  • Navegadores, servidores web, hosting y dominios
  • Herramientas clave: Node.js, NPM, Git, Visual Studio Code
  • Control de versiones con Git y GitHub
  • Integración y despliegue continuo (CI/CD) con GitHub Actions

2. Interfaces Web Modernas y Accesibles

  • HTML5 y estructura semántica
  • CSS3, modelo de caja, media queries
  • Bootstrap/Tailwind CSS: estilos y componentes reutilizables
  • Principios de accesibilidad (WCAG)
  • Diseño adaptativo y responsive

3. Patrones de Diseño para la Arquitectura del Back-End

  • Patrones Creacionales: Singleton, Factory Method
  • Patrones Estructurales: Adapter, Decorator, Composite
  • Patrones de Comportamiento: Observer, State

4. Desarrollo Front-End con Frameworks

  • Introducción a Angular
  • Angular CLI: componentes, templates, módulos y servicios
  • JSON y métodos HTTP (GET, POST, PUT, DELETE)
  • Creación y consumo de servicios desde Angular usando async/await
  • Firebase y Backend-as-a-Service (BaaS)
  • Formularios reactivos y validaciones
  • Ruteo y navegación entre vistas

Prácticas

  • Instala Node.js, Git y Visual Studio Code
  • Crea una página de presentación personal usando etiquetas semánticas
  • Crea un menú adaptable para dispositivos móviles usando Tailwind o Bootstrap
  • Mejora de accesibilidad: añade etiquetas ARIA, alt, label y asegura navegación con teclado
  • Crea una función createUser(type) usando el patrón Factory Method para crear diferentes tipos de usuarios
  • Usa el patrón Adapter para adaptar una API simulada
  • Crea un sistema de "notas" que permita añadir estilos con el patrón Decorator
  • Usando Angular CLI crea al menos 3 componentes y un servicio para compartir datos
  • Consume una API pública usando HttpClient e implementa métodos GET, POST, PUT y DELETE
  • Crea un formulario de registro con validaciones reactivas (nombre, email, contraseña)
  • Implementa ruteo entre las vistas del formulario, inicio y dashboard

Proyecto de Asignatura

El proyecto integrador deberá demostrar el desarrollo y alcance de las competencias de la asignatura, considerando las siguientes fases:

1. Fundamentación

Marco referencial (teórico, conceptual, contextual y legal) basado en un diagnóstico que permita comprender la realidad objeto de estudio.

2. Planeación

Diseño del proyecto con asesoría del docente, planificando el proceso de intervención, actividades, recursos y cronograma de trabajo.

3. Ejecución

Desarrollo de la planeación del proyecto, fase de mayor duración que implica el desempeño de las competencias genéricas y específicas.

4. Evaluación

Fase final que aplica un juicio de valor, promoviendo la evaluación para la mejora continua, metacognición y pensamiento crítico.

Evaluación por Competencias

  • Prácticas con herramientas de desarrollo
  • Examen escrito
  • Proyecto integrador que ponga en práctica los conocimientos de la asignatura
  • Participación en eventos académicos (congresos o concursos)
  • Investigación documental
  • Rúbricas para la evaluación de las prácticas
  • Utilización de herramientas de desarrollo y programación
